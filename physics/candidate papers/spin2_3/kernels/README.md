# Kernels

Kernel files are the domain-level working documents for the framework. They are more detailed than the compendium and less speculative than research-track notes.

## Files

| File | Role |
| --- | --- |
| [statics.md](statics.md) | Representation structure, static inputs, octonionic internal structure |
| [dynamics.md](dynamics.md) | Reduced dynamics, two-branch transport, scaling laws |
| [epistemics.md](epistemics.md) | Observability, projection, hidden-sector access |
| [consistency.md](consistency.md) | Algebraic, dynamical, gauge, and selection consistency |
| [interpretation.md](interpretation.md) | Safe and stronger interpretive readings |
| [phenomenology.md](phenomenology.md) | Candidate observables and prediction-readiness |
| [topological.md](topological.md) | Tenfold/DIII/topological transition track |
| [open-problems.md](open-problems.md) | Proof debts, blockers, and priority order |

## Claim Level

Kernel files may identify Level 5 conjectural directions, but strong claims should be mirrored in [../CLAIM_LEDGER.md](../CLAIM_LEDGER.md) before they become paper-facing.
