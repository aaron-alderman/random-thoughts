# Kernels

Kernel files are the domain-level working documents for the framework. They are more detailed than the compendium and less speculative than research-track notes.

## Files

| File | Role |
| --- | --- |
| `statics.md` | Representation structure, static inputs, octonionic internal structure |
| `dynamics.md` | Reduced dynamics, two-branch transport, scaling laws |
| `epistemics.md` | Observability, projection, hidden-sector access |
| `consistency.md` | Algebraic, dynamical, gauge, and selection consistency |
| `interpretation.md` | Safe and stronger interpretive readings |
| `phenomenology.md` | Candidate observables and prediction-readiness |
| `topological.md` | Tenfold/DIII/topological transition track |
| `open-problems.md` | Proof debts, blockers, and priority order |

## Stability Rule

Kernel files may identify conjectures, but strong claims should be mirrored in `../CLAIM_LEDGER.md` before they become paper-facing.
