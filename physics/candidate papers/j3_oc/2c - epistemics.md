# Epistemics Kernel

## Purpose

This document is the epistemic and observable source text for the `j3_oc` branch.

Its job is to say:

- what kinds of objects count as states after reduction
- what counts as an observable
- where measurement or access lives in the formalism
- how the observable sector arises from the ambient branch

This file is one of the main reasons the branch exists. Without it, the ambient branch stays architecturally rich but physically loose.

---

## Scope

This file covers:

- candidate state-space language after reduction
- candidate observable language after reduction
- measurement as access to the reduced observable sector
- the difference between ambient existence and observable access

This file does not cover:

- a full quantum measurement theory
- empirical phenomenology
- a fully derived reduced dynamics

---

## Core epistemic question

The central question is:

> once the ambient complexified Jordan branch has been reduced, what exactly is visible?

This branch should not assume that ambient presence implies direct observation.

Instead it should distinguish:

- ambient structure
- reduced admissible structure
- observable structure

That distinction is central to the whole `j3_oc` project.

---

## Provisional state / observable / measurement triad

The branch will use the following provisional language.

- `states =` admissible reduced Jordan-sector data, such as selected elements, orbit representatives, or positive/Hermitian classes in the reduced branch
- `observables =` Jordan elements or functions on the reduced state space that survive the observable-sector restriction
- `measurement =` evaluation, expectation, or readout on the observable slice after the ambient-to-observable reduction has been applied

This is not yet a finished theorem. It is a necessary anchor sentence so the branch has a clear observable grammar.

---

## Observable access

The key epistemic proposal of this branch is:

- observable access is downstream of reduction

In other words:

- the ambient branch may be larger than what is observed
- observability is not assumed to coincide with ambient existence
- the observable sector is what remains after admissibility, reduction, and access restrictions are imposed

This fits the chain:

`A_ambient -> S_relevant -> S_reduced -> S_observable`

The epistemic layer lives primarily at the last arrow, but depends on the earlier ones.

---

## What this branch owes beyond `spin2_3`

The reduced `spin2_3` branch already poses a clear question about why one channel becomes observable.

The `j3_oc` branch owes a broader answer:

- why a reduced observable slice should arise at all from the complexified ambient object
- what structures are hidden, filtered, quotient-out, or coarse-grained
- whether observability is defined by positivity, selected interactions, or allowable readout functionals

Until that is made sharper, `j3_oc` remains an ambient organizer more than an observable theory.

---

## Main epistemic options

The main live possibilities are:

1. observability is defined by positivity-compatible reduced states
2. observability is defined by a chosen interaction channel after reduction
3. observability is defined by a distinguished observable algebra inside the reduced sector
4. observability is defined by coarse-graining over ambient redundancies

The branch does not yet choose one of these as fully canonical.

The safe statement is:

- observable access is a branch burden that must be made explicit rather than assumed

---

## Safe epistemic statements

The branch can safely say:

1. ambient size and observable access must be separated
2. states, observables, and measurement need explicit post-reduction definitions
3. measurement in this branch is most safely read as readout on the observable slice, not direct access to the whole ambient arena

The branch cannot yet safely say:

- that it has a full measurement theory
- that the observable slice is uniquely determined
- that the same access rule as `spin2_3` already follows from the complexified ambient object

## Major unresolved issues

The epistemic domain still owes:

1. a sharper definition of admissible reduced states
2. a sharper definition of the observable algebra
3. a sharper account of what operation counts as measurement
4. a more precise statement of why the observable slice is privileged

These are major obligations, not optional refinements.

---

## Working bottom line

The epistemic spine of `j3_oc` is:

1. the ambient branch is larger than the observable branch
2. observability lives after reduction, not before it
3. the branch now has explicit provisional language for states, observables, and measurement
4. the exact observable rule remains open and must be earned later

That is already a meaningful improvement in branch discipline.
