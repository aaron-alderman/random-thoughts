# Consistency Kernel

## Purpose

This document is the consistency source text for the `j3_oc` branch.

Its job is to say:

- what the complexified branch must earn to count as coherent
- what would count as a genuine advantage over `J3(O)`
- where the strongest proof burdens live

This file is where "larger" must become either "better controlled" or "not yet justified."

---

## Scope

This file covers:

- admissibility and reduction discipline
- reality and Hermiticity burdens
- uniqueness and non-uniqueness questions
- what would count as a genuine gain over `J3(O)`
- whether reduced `spin2_3` structure can be obtained without ad hoc inserts

This file does not cover:

## Core consistency question

The central question is:

> once `J3(C \otimes O)` is chosen as the ambient object, what is actually forced, what is merely convenient, and what remains unresolved?

This question is sharper here than in the real branch because the complexified branch carries a larger proof burden.

---

## Main consistency burdens

### C1. Admissibility burden

The branch must justify which sectors of the ambient complexified object count as physically admissible.

If that is done only by selective convenience, the branch weakens immediately.

The current best consistency posture is:

- start with Hermitian or reality-compatible admissibility
- sharpen with rank or orbit control
- treat that combined path as the active strong admissibility claim
- keep pure Hermitian-first available only as a weaker fallback
- keep stabilizer-sensitive conditions alive in parallel only if they can be defended without reverse-engineering the conclusion

So the current branch default should be judged by a simple rule:

- if the stabilizer-sensitive refinement does not sharpen `u`, `SU(3)`, or reduced-branch selection, it has probably not earned its place

### C2. Reality / Hermiticity burden

The branch must explain what role real, Hermitian, or positivity-compatible restrictions play.

If those restrictions do all the real physical work, then the branch must explain why the real `J3(O)` branch was not already enough.

### C3. Reduction-discipline burden

The branch must show that the path from ambient object to reduced effective branch uses named operations rather than selective matching.

This is the central consistency issue of the folder.

### C4. Comparative burden against `J3(O)`

The branch must earn something the real branch does not already give more simply.

If it does not, then the real branch remains competitive or preferable.

---

## What would count as a genuine advantage over `J3(O)`

The complexified branch would earn a real consistency advantage if it can do one or more of the following:

1. define a cleaner ambient-to-observable reduction
2. improve the status of `u` from bare choice toward induced structure
3. narrow the gap between structural `SU(3)` and physical color
4. give a more controlled observable grammar
5. obtain a reduced `spin2_3`-type branch without selective inserts

Those would count as substantive gains.

What would **not** count by itself:

- having more room
- allowing more interpretations
- reproducing reduced structures only after many unconstrained fixes

---

## Uniqueness and non-uniqueness

The branch should separate at least three levels:

1. a structure exists in the ambient complexified arena
2. the structure is selected by disciplined reduction
3. the structure is uniquely forced by the branch

This matters for:

- `u`
- `SU(3)`
- observable access
- reduced `Spin(2,3)` structure

The branch is currently strongest only at level `1`, and is trying to improve levels `2-3`.
